## 文档信息

- **产品名称**：学科小勇士
- **版本**：V1.0
- **设计日期**：2026-06-26
- **对应 PRD**：[儿童学科闯关游戏产品需求文档](../../../../outputs/儿童学科闯关游戏产品需求文档.md)
- **设计状态**：待实现

---

## 1. 设计目标

将小学 4-5 年级语文、数学、英语三门学科的学习内容，包装成“升级打怪”的冒险游戏。通过答题战斗、角色成长、家庭奖励池、周末抽奖等机制，让孩子主动学习。本产品为家庭内部使用，不面向商业化，数据本地存储，家长掌握全部管理权限。

## 2. 约束与决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| 实现范围 | PRD 完整版一体化实现 | 家长希望一次性覆盖所有功能 |
| 技术栈 | React 19.x + Vite + Tailwind CSS | 现代、高效、PWA 支持好 |
| 状态管理 | Zustand | 轻量，适合家庭项目 |
| 本地存储 | IndexedDB（via `idb` 包） | 容量大、结构化、支持复杂查询 |
| 路由结构 | `/play` 孩子端 + `/parent` 家长端 | 两个独立入口，共享同一份数据 |
| AI 出题 | 前端直接调用云端大模型 API | 实现简单，家庭自用风险可控 |
| API Key 存储 | 保存在 IndexedDB，前端配置 | 无需后端，符合纯本地架构 |
| 离线能力 | Service Worker + IndexedDB 缓存 | 联网时预生成题库，离线可玩 |
| 美术风格 | 纯 CSS/HTML 卡通风格 | 无需外部素材，加载快 |
| 家长设置 | 可视化配置 | 降低使用门槛，提升体验 |

## 6. AI 出题与离线缓存

### 6.1 出题流程

1. 家长在 `/parent` 进入“AI 出题”页面
2. 选择学科、知识点、难度、题目数量
3. 点击“生成题库”，携带 API Key 调用大模型 API
4. 解析返回 JSON，校验格式后写入 `questions` store
5. 显示生成结果：成功数量、失败数量、耗时

### 6.2 Prompt 模板结构

每个学科独立 prompt，要求模型严格返回 JSON：

```
你是一位小学{grade}年级{subject}出题老师。
请根据以下知识点生成 {count} 道选择题：
知识点：{topics}
难度：{difficulty}（1简单，2中等，3困难）

要求：
1. 每道题有 4 个选项，只有一个正确答案
2. 题目适合小学{grade}年级学生
3. 返回严格 JSON 格式
4. 包含答案解析

返回格式：
{
  "questions": [
    {
      "question": "题干",
      "options": ["A", "B", "C", "D"],
      "answer": 0,
      "explanation": "解析"
    }
  ]
}
```

### 6.3 离线缓存策略

- 应用静态资源：Service Worker 缓存，离线可用
- 题目数据：生成后写入 IndexedDB，闯关时从本地读取
- 用户进度：实时写入 IndexedDB
- 联网检测：启动时检查网络，联网状态下检测题库余量，不足时提醒家长补充

### 6.4 异常处理

- API Key 无效：提示家长检查 Key
- 网络超时：允许重试，或切换为本地已有题库
- 返回格式错误：展示原始响应，方便调试 prompt

## 7. 核心游戏系统

### 7.1 世界地图

- 三大区域：语文之森（绿色系）、数学迷宫（蓝色系）、英语海岸（黄色系）
- 每个区域 8-12 个普通关卡 + 1 个区域 Boss
- 关卡状态：已解锁、已通过、未解锁

### 7.2 战斗流程

```
进入关卡 → 显示怪物/Boss → 展示题目 → 限时答题
    ↓
答对 → 角色攻击 → 触发连击/暴击判定 → 怪物血量减少
    ↓
答错/超时 → 怪物反击 → 护盾减少 → 护盾归零则失败
    ↓
怪物血量归零 → 通关结算（星星、经验、碎片）
```

### 7.3 战斗数值

| 属性 | 说明 | 计算方式 |
|---|---|---|
| 基础伤害 | 答对题目造成的伤害 | 10 + 等级 × 2 |
| 连击加成 | 连续答对提升伤害 | 第 3 题起伤害 ×1.5，第 5 题起 ×2 |
| 暴击 | 连续 3 题答对后下一次暴击 | 伤害 ×2 |
| 护盾 | 可承受的错误次数 | 3 + floor(等级 / 5) |
| 怪物血量 | 随关卡难度提升 | 基础 30 × 难度系数 |

### 7.4 角色成长

| 等级 | 称号 | 解锁内容 |
|---|---|---|
| 1-5 | 学习小新手 | 基础地图、初始宠物 |
| 6-15 | 知识探索者 | 新装扮、第二只宠物 |
| 16-30 | 学科小勇士 | 高级装备、团队玩法 |
| 31-50 | 智慧守护者 | 终极称号、限定皮肤 |

升级经验公式：`nextLevelExp = level × 100`

### 7.5 装备系统

| 装备位 | 作用 |
|---|---|
| 武器 | 增加基础伤害 |
| 盾牌 | 增加护盾值 |
| 法杖 | 增加暴击率 |
| 鞋子 | 增加答题时间上限 |

- 装备通过碎片合成
- 碎片来源：关卡奖励、成就、赛季活动

### 7.6 宠物系统

- 宠物陪伴战斗，拥有被动技能
- 宠物可进化，进化条件绑定学习行为：
  - 连续 3 天完成数学练习 → 一阶进化
  - 累计答对 100 道英语题 → 二阶进化
  - 完成期末复习 → 终极形态
- 宠物技能：提示一次、排除错误选项、回血、双倍星星

### 7.7 错题本

- 答错题目自动加入 `wrongQuestions`
- 错题按学科分类，显示答错次数
- 每周五弹出“错题 Boss 挑战”
- 错题答对后从错题本移除或减少计数

## 8. 奖励与经济系统

### 8.1 星星获取

| 行为 | 星星奖励 |
|---|---|
| 答对普通题 | 1-2 颗 |
| 答对难题 | 3 颗 |
| 连续答对 3 题暴击 | 额外 3 颗 |
| 完成每日任务 | 5-10 颗 |
| 完成错题复习 | 3 颗 |
| 连续打卡 7 天 | 20 颗 |
| 升级 | 50 颗 |
| 家长自定义任务 | 家长设定 |

### 8.2 星星消耗

- 家庭奖励池兑换
- 周末抽奖券：10 颗/张
- 虚拟商城购买装扮、宠物饲料、家具

### 8.3 每日上限

- 默认每日星星获取上限 100 颗
- 家长可在 0-300 颗之间调节
- 达到上限后继续答题不奖励星星，但仍记录成长和成就

### 8.4 家庭奖励池

- 家长通过可视化界面添加奖励
- 奖励字段：名称、分类、星星价格、库存、说明、图标
- 默认模板：肯德基一次、麦当劳一次、必胜客一次、晚睡券、电影选择权、免家务券、周末出游等
- 兑换流程：孩子申请 → 扣除星星 → 家长确认 → 现实中兑现

### 8.5 周末抽奖

- 每周六/日 19:30-20:00 开放
- 孩子用 10 颗星星购买抽奖券
- 奖品池包括：小星星、虚拟道具、特权卡、实物兑换碎片、惊喜大奖、谢谢参与
- 保底机制：每 10 抽必得 1 个实物兑换碎片，5 个碎片合成一次餐饮兑换

### 8.6 虚拟商城

- 商品类型：装扮、宠物皮肤、特效、基地家具
- 购买后立即生效，无需家长确认
- 部分限定商品通过成就解锁

## 10. 非功能性需求

### 10.1 PWA 能力

- `manifest.json`：应用名称、图标、主题色、启动方式
- Service Worker：缓存静态资源，支持离线访问
- 可安装到手机/平板桌面

### 10.2 健康使用

- 每 20 分钟护眼提醒，休息 1 分钟
- 每日答题时长上限默认 45 分钟，家长可调
- 连续答题 45 分钟强制休息 5 分钟
- 晚上 21:00 后进入休息模式
- 达成护眼目标获得“护眼小卫士”成就

### 10.3 安全与隐私

- 无账号系统，不收集个人信息
- API Key 仅本地存储
- 数据导出为 JSON，不上传第三方
- 家长端可选设置简单 PIN

### 10.4 数据备份

- 家长端支持“导出全部数据”为 JSON
- 支持“导入数据”恢复
- 建议每月备份一次

### 10.5 浏览器兼容性

- 主要支持 Chrome、Edge、Safari 最新版
- Firefox 基本支持
- 不支持 IE

## 12. 风险与注意事项

1. **API Key 安全**：Key 存储在前端，虽然家庭自用风险可控，但仍建议设置用量限制，避免 Key 泄露后被滥用。
2. **AI 题目质量**：生成题目需人工抽查，确保无错误、无超纲、无歧义。
3. **数据丢失**：IndexedDB 可能因浏览器清理而丢失，必须定期导出备份。
4. **防沉迷执行**：时间限制需要在前端强制生效，孩子可能尝试刷新页面绕过，需结合 Service Worker 做基础校验。
5. **奖励物质化**：避免把所有生活奖励都和学习绑定，保持家庭互动的非功利性。

## 14. 结论

本设计采用 React 19.x + Vite + Tailwind + IndexedDB 的一体化 PWA 架构，通过 `/play` 和 `/parent` 两个入口分别服务孩子和家长。AI 出题在联网时批量生成并本地缓存，支持离线游玩。完整版涵盖三学科闯关、角色成长、装备宠物、家庭奖励池、周末抽奖、成就图鉴、家长管理等 PRD 全部功能。
