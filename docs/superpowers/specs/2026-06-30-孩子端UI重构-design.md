# 学科小勇士 — 孩子端 UI 重构设计文档

## 文档信息

- **产品名称**：学科小勇士
- **版本**：V3 UI 风格升级
- **设计日期**：2026-06-30
- **设计状态**：待实现
- **参考原型**：[首页效果图](../../../../outputs/boys-game-playhome-mockup-v1.png)

---

## 1. 设计目标

把孩子端 `/play` 的整体视觉从现有暗色“星球太空”主题升级为更明亮、更符合儿童手游认知的“扁平游戏关卡风”。核心目标：

1. 让孩子一眼找到“开始闯关”主行动点。
2. 把常用功能（商城、副本、错题本）固定到底部 Tab，降低返回成本。
3. 把次常用功能（装备、宠物、成就、任务、抽奖、设置）放到首页两侧快捷栏，保持首页简洁。
4. 所有孩子端页面沿用同一套组件与配色，避免风格割裂。

---

## 2. 约束与决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| 整体风格 | 扁平游戏关卡风 | 符合手绘稿布局，儿童熟悉手游主界面 |
| 背景方向 | 明亮渐变天空/草地 | 长时间使用更舒适，突出中央关卡卡片 |
| 头像方案 | Emoji + 游戏徽章边框 | 无需外部图片资源，点击可换已解锁头像 |
| 主导航 | 底部 4 Tab：营地/商城/副本/错题本 | 手绘稿明确的四个高频入口 |
| 次导航 | 首页两侧圆形图标列 | 手绘稿右侧“背包”思路的延伸 |
| 范围 | 所有 `/play` 页面 | 保证整体风格统一 |
| 实现方式 | 纯 Tailwind CSS + Lucide 图标 | 与现有技术栈一致，不引入新依赖 |

---

## 3. 视觉设计系统

### 3.1 色彩

- **主背景**：`#e0f2fe` → `#dcfce7` 线性渐变（天空到草地），柔和明亮。
- **主按钮**：橙色渐变 `#f97316` → `#fb923c`，适合行动召唤。
- **主卡片**：白色 `#ffffff`，带轻微投影 `shadow-lg`。
- **学科色沿用**：
  - 语文绿 `chinese-500` (#22c55e)
  - 数学蓝 `math-500` (#3b82f6)
  - 英语黄 `english-500` (#eab308)
- **状态色**：成功绿、错误红、锁定灰 `#9ca3af`。

### 3.2 字体与字号

- 使用系统默认无衬线字体，不引入外部字体。
- 页面标题：`text-xl font-bold`
- 卡片标题：`text-lg font-bold`
- 正文/标签：`text-sm`
- 数值：`text-base font-bold`

### 3.3 圆角与阴影

- 大卡片：`rounded-2xl`
- 按钮：`rounded-xl`
- 图标按钮：`rounded-full`
- 胶囊条：`rounded-full`
- 投影：`shadow-md` 到 `shadow-lg`，不使用大模糊阴影。

### 3.4 通用组件

- **头像徽章**：圆形，白色边框 + 主题色描边，右下角显示等级数字。
- **状态胶囊**：圆角长条，左侧图标 + 右侧数值，例如体力、星星。
- **章节节点**：圆形学科色节点，已完成实心、当前发光、锁定灰色。
- **图标按钮**：48px 圆形，白色底，图标用学科色或中性灰。
- **底部 Tab**：胶囊形按钮，选中项上浮并填充主题色。

## 4. 首页（营地）布局

参考原型图，首页自上而下分为五个区域。

### 4.1 顶部状态条

- **左侧**：头像徽章，显示当前 emoji 头像和等级。点击后弹出“个人设置”面板，可更换已解锁头像/昵称。
- **右侧两个胶囊**：
  - 体力：黄色闪电图标 + `当前值/最大值`
  - 星星：黄色星星图标 + 总星星数

### 4.2 中央章节卡

- 白色大圆角卡片，视觉重心。
- 标题：`第 N 章（主题名）`，例如“第1章（拼音冒险）”。
- 三科进度轨道：一条微弯的虚线连接三个节点：
  - 语文节点：绿色
  - 数学节点：蓝色
  - 英语节点：黄色
- 节点状态：
  - 已通关：实心填充 + 对勾
  - 当前可挑战：发光脉冲
  - 未解锁：灰色空心

### 4.3 主行动按钮

- 大号橙色渐变按钮：“开始闯关”。
- 点击直接进入当前推荐关卡（统一关号逻辑）。

### 4.4 两侧快捷入口

- **左侧竖列**（从上到下）：
  1. 装备
  2. 宠物
  3. 成就
- **右侧竖列**（从上到下）：
  1. 背包
  2. 每日任务
  3. 抽奖
  4. 设置/个人
- 每个入口为 48px 圆形图标按钮，带轻微投影和悬停放大。

### 4.5 底部 Tab 栏

- 四个一级入口：营地、商城、副本、错题本。
- Tab 样式：
  - 未选中：透明底 + 灰色图标文字
  - 选中：白色填充底 + 主题色图标，按钮略微上浮。
- Tab 栏本身为圆角浮岛，底部留出安全间距。

## 5. 子页面设计规范

所有子页面沿用同一套视觉系统：顶部标题栏 + 圆角卡片内容区。

### 5.1 顶部标题栏

- 左侧：返回箭头，返回 `/play` 或上一页。
- 中间：页面标题。
- 右侧：星星 + 体力胶囊（与首页一致）。

### 5.2 商城（/play/shop）

- 顶部分类 Tab：装备、宠物、装扮、家具。
- 商品卡片网格：每个卡片显示物品图标、名称、星星价格、购买按钮。
- 已拥有的物品显示“已拥有”标签。

### 5.3 副本（/play/arcade）

- 小游戏入口卡片列表：五子棋、百科问答、记忆翻牌、速算挑战、词语接龙。
- 每个卡片带学科色标签和一句话玩法说明。

### 5.4 错题本（/play/wrong）

- 顶部分科筛选标签：语文、数学、英语。
- 错题卡片列表：显示题目摘要、答错次数、上次答错时间。
- 卡片右侧“去复习”按钮。

### 5.5 战斗（/play/battle/:subject/:levelNumber）

- 保留英雄/怪物对峙布局。
- HP 条改为圆角胶囊，带颜色区分（英雄蓝/绿，怪物红/紫）。
- 题目和选项使用大圆角卡片。
- 正确反馈：绿色脉冲 + 对勾。
- 错误反馈：红色抖动 + 叉号。
- 连击/得分使用弹出数字动画。

### 5.6 成就（/play/achievements）

- 成就徽章网格。
- 已解锁徽章彩色高亮，未解锁徽章灰色并显示解锁条件。

### 5.7 每日任务（/play/tasks）

- 任务列表卡片：任务描述、进度条、奖励星星数、领取按钮。

### 5.8 抽奖（/play/lottery）

- 中央抽奖转盘或抽奖卡片堆。
- 底部显示持有抽奖券数量和“抽一次”按钮。

### 5.9 装备（/play/equipment）

- 上方四个装备槽（武器、盾牌、法杖、鞋子）呈圆形或半圆排列。
- 下方背包网格，显示可装备物品。

### 5.10 宠物（/play/pet）

- 中央宠物头像和亲密度进度条。
- 下方操作卡片：喂食、玩耍、进化。

### 5.11 个人设置/档案

- 头像选择网格：只显示已解锁头像，未解锁头像显示锁和解锁条件。
- 昵称输入框，确认后保存到 `profileStore`。
- 底部“切换家长端”入口（可选，保留现有逻辑）。

## 6. 交互与动画

- **页面切换**：保留现有路由过渡，可加入轻微左右滑入。
- **按钮反馈**：点击时缩放 `scale-95`，释放后回弹。
- **节点脉冲**：当前可挑战学科节点使用 `animate-pulseGlow` 或类似发光动画。
- **星星/体力变化**：数值变化时做弹跳数字动画。
- **Tab 切换**：选中项上浮 `translate-y-[-4px]` + 填充色变化。
- **加载状态**：使用旋转的星球/星星图标，符合儿童期待。

---

## 7. 响应式与适配

- 以手机竖屏为主，最大宽度 `max-w-md` 居中显示。
- 两侧快捷入口在更窄屏幕上可缩小至 40px。
- 底部 Tab 在横屏或平板时保持等分，不拉伸变形。

---

## 8. 实现注意事项

- 不引入新的图片资源或 UI 组件库，全部用 Tailwind + Lucide + emoji 实现。
- 保持现有 `profileStore` 和路由结构，仅调整组件样式和布局。
- 底部 Tab 需要新增 `BottomTabBar` 组件，并注入到 `PlayLayout` 的指定一级页面中。
- 两侧快捷入口只在首页 `/play` 显示，子页面隐藏。
- 头像/昵称设置需要新增一个弹窗或独立页面，避免直接修改家长端设置。
