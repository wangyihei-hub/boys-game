# 学科小勇士 V2.0 迭代需求规划

## 文档信息

- **产品名称**：学科小勇士
- **迭代版本**：V2.0
- **规划日期**：2026-06-27
- **基于版本**：V1.0（Phase 6 已完成）
- **迭代目标**：以人教版四年级→五年级暑假 90 天通关为主线，重构 UI 为沉浸式游戏体验，新增 5+ 副本玩法

---

## 1. 迭代背景与目标

### 1.1 当前版本问题

V1.0 版本已完成基础功能框架，但存在以下核心问题：

1. **出题模式被动**：题目需要家长手动触发 AI 生成，流程繁琐，实际使用中家长往往不会主动出题，导致孩子进入关卡后只能做种子题（数量不足），体验不完整
2. **关卡内容空泛**：当前 3 个学科区域各只有 6 个关卡（共 18 关），知识点没有与真实教材对齐，无法形成系统性学习路径
3. **UI 过于朴素**：纯 Tailwind + 简单 CSS 动画，缺乏游戏感，孩子玩几次后容易觉得枯燥
4. **玩法单一**：除了主线战斗（做题）外，仅有抽奖和每日任务，缺乏多样性的娱乐玩法

### 1.2 迭代目标

1. **自动化主线出题**：系统启动时自动生成完整题库，按人教版教材编排 90 天通关计划，家长无需手动干预
2. **教材级关卡设计**：三大学科 × 90 天，每个关卡对应具体知识点，复习四年级 + 预习五年级，形成完整暑假学习闭环
3. **沉浸式游戏 UI**：参考《一步两步》等休闲游戏的动画风格，粒子特效、场景切换、角色成长动效，让孩子有"真的在玩游戏"的感觉
4. **多元副本玩法**：至少 5 种非战斗玩法（五子棋、百科常识等），作为主线的调剂和奖励，降低学习疲劳

---

## 2. 主线任务：90 天暑假通关

### 2.1 核心理念

将小学 4-5 年级暑假 90 天设计为"冒险旅程"：每天 1-3 个关卡，覆盖语文、数学、英语三科。前 30 天复习四年级，中 30 天过渡衔接，后 30 天预习五年级。

### 2.2 阶段划分

| 阶段 | 天数 | 主题 | 学科侧重 | 难度范围 |
|---|---|---|---|---|
| 第一阶段：知识回顾 | Day 1-30 | 复习四年级上下册 | 三科均衡 | 1-2 |
| 第二阶段：能力提升 | Day 31-60 | 四年级拔高 + 五年级衔接 | 三科均衡 | 2-3 |
| 第三阶段：五年级预演 | Day 61-90 | 预习五年级上册 | 三科均衡 | 2-3 |

### 2.3 每日节奏设计

每天设计 2-3 个关卡（三科轮换），每个关卡包含 3-5 道题目：

| 时间段 | 内容 | 时长预估 |
|---|---|---|
| 上午 | 数学关卡（逻辑思维最佳时段） | 5-8 分钟 |
| 下午 | 语文关卡（阅读+记忆） | 5-8 分钟 |
| 傍晚 | 英语关卡（听力+词汇） | 5-8 分钟 |

每日总时长控制在 15-25 分钟，配合护眼提醒（每 20 分钟休息 1 分钟）。

### 2.5 关卡数据结构

每个关卡定义：

```typescript
interface Stage {
  id: string;              // 如 "math-d01-s1"
  day: number;             // 第几天（1-90）
  subject: Subject;        // chinese | math | english
  name: string;            // 关卡名，如 "大数的认识·读写"
  knowledgePoints: string[]; // 涉及的知识点
  difficulty: Difficulty;  // 1 | 2 | 3
  questionCount: number;   // 3-5 题
  isBoss: boolean;         // 是否为 Boss 关
  isCheckpoint: boolean;   // 是否为阶段检查点
  unlockCondition: string; // 解锁条件（前置关卡 ID）
  rewards: {               // 通关奖励
    stars: number;
    exp: number;
    fragments?: string;    // 碎片类型
  };
}
```

### 2.6 自动出题策略

**取消家长手动出题**，改为系统自动管理：

1. **首次启动**：调用 AI 模型，按 90 天规划批量生成题目（每关 5 题 × 270 关 ≈ 1350 题），存入 IndexedDB
2. **每日补充**：启动时检查当日及后续 3 天的题目是否充足，不足则自动生成
3. **四年级复习发散延展**：第一阶段（Day 1-30）的四年级复习题目不能停留在"直接回忆"，必须通过 **横向发散**（同知识点不同变体、跨知识点串联）和 **纵向拔高**（从识记到理解再到应用）来增加思考深度。例如数学复习"大数的认识"时，不只考读数和比大小，还应穿插实际场景应用题；语文复习"修辞手法"时，不只考判断，还应要求仿写和赏析
4. **错题复练**：答错的题目自动加入复习队列，3 天后再次出现
5. **本地兜底**：如果 AI 调用失败，使用本地出题器（`localQuestionGenerator`）生成替代题目，确保不卡关

家长端保留"查看题库""手动补充"功能，但不再是必需步骤。

### 2.7 关卡通过判定逻辑

**核心问题**：如果关卡都是选择题，孩子可以反复尝试、"蒙"出正确答案，完全绕开学习目的。需要从四个层面构建防作弊体系。

#### 2.7.1 第一层：题库防重复 — 每次看到不同的题

每个知识点关卡对应的不是固定 5 道题，而是一个 **15-30 题的题库**。

- 每次进入关卡时，从题库中随机抽取 3-5 题
- 前后两次抽取的题目交集尽量控制在 1 题以内（用上次抽取的题目 ID 做剔除）
- 首次进入关卡时不展示"已答过的题"
- 孩子即使重试 3 次，总共也只接触约 12-15 题，远不会刷穿题库

题库由 AI 批量生成，按知识点分组存入 IndexedDB，每组题目数量 ≥ 15。

#### 2.7.2 第二层：题型防蒙 — 不能全凭运气

当前 V1.0 题型以 choice 为主，fillblank 占比低。V2.0 强制每个关卡 5 题的题型配比为：

| 题型 | 数量 | 说明 |
|---|---|---|
| `choice`（选择题） | 2 道 | 4 选 1，答案索引 0-3 |
| `fillblank`（填空题） | 2 道 | 数学计算得出数值、语文默写字词、英语拼写单词 |
| `spelling`（拼写/默写题） | 1 道 | 语文古诗默写、英语单词拼写 |

**为什么这套配比防蒙？**

- choice 每题蒙对概率 25%，蒙对 2 题的概率仅 6.25%，但即使全蒙对也只贡献 2 题
- fillblank 和 spelling 完全无法蒙——数学要算对得数、语文要写对字词、英语要拼对单词
- 通关条件至少需要答对 3/5 题（正确率 ≥ 60%），必须在填空题上展现真实能力
- 同一道选择题每次进入时 **四个选项随机乱序**，即使记住了"上次选第三个"也无效

#### 2.7.3 第三层：失败惩罚 — 重试有代价

| 失败次数 | 惩罚 | 说明 |
|---|---|---|
| 第 1 次 | 扣除 1 颗星星 + 30 分钟冷却 | 可消耗 5 颗星星跳过冷却 |
| 第 2 次 | 扣除 2 颗星星 + 60 分钟冷却 | 可消耗 10 颗星星跳过冷却 |
| 第 3 次及以上 | 扣除 3 颗星星 + **强制跳转错题复习** | 必须先做对之前所有错题才能继续主线 |

设计要点：
- 冷却期内可以玩副本玩法（五子棋、百科等），不浪费时间
- 每次重试时，如果某道题已在之前的尝试中答错过，本次再答错会承受 **双倍伤害**（怪物攻击力 ×2）
- 星星不足 3 颗时无法支付重试费用，必须等待冷却结束

#### 2.7.4 第四层：行为防刷 — 乱点惩罚

防止孩子不读题"快速乱点"：

| 行为模式 | 判定条件 | 惩罚 |
|---|---|---|
| 疑似乱点 | 答题时间 &lt; 2 秒且答错 | 伤害翻倍，本题不参与连击计数 |
| 确认乱点 | 答题时间 &lt; 1 秒 | 无论对错，伤害减半，暴击计数清零 |
| 犹豫修改 | 5 秒内修改过答案 | 最终提交时伤害减半（鼓励先想好再选） |
| 连续多题 &lt; 3 秒 | 连续 3 题答题时间均 &lt; 3 秒 | 强制弹出提示"请认真读题后再作答"，并暂停 5 秒 |

注意：行为检测只在 **choice 题型**上执行，fillblank 和 spelling 本身需要输入答案，不存在乱点问题。

#### 2.7.5 通关 vs 失败 vs 重试流程

```
进入关卡 → 展示怪物 → 逐题作答
    │
    ├─ 答对 ≥ 3/5 题 → 怪物血量归零 → 🌟 通关结算
    │
    ├─ 答对 2/5 题 → 怪物血量还有剩余 → ⚠️ 当前进度保存，可选择：
    │       ├─ 用 3 颗星星买"复活" → 继续战斗
    │       └─ 放弃 → 视为失败
    │
    └─ 答对 ≤ 1/5 题 → ⚔️ 直接失败
                └─ 失败惩罚（扣星+冷却）
                └─ 错题自动转入复习队列
                └─ 冷却结束后可重试，题库重新随机抽取
```

#### 2.7.6 预期效果

| 孩子行为 | 系统反制 | 最终结果 |
|---|---|---|
| 想蒙选择题 | 题库防重复 + 选项随机乱序 + 填空题无法蒙 | 蒙对的概率极低，且不足以通关 |
| 反复重试刷题 | 冷却时间 + 扣星 + 连败强制错题复习 | 要么等，要么花星星，要么先复习再回来 |
| 不读题乱点 | 短时答题检测 + 伤害惩罚 + 连击清零 | 伤害不够打不死怪物，必然失败 |
| 正常答题 | 无额外惩罚 | 一次通过率与真实知识水平正相关 |

这套机制的本质是 **把"靠蒙过关"变成比"认真答题一次通过"更慢、更贵、更累的选择**，从而让孩子主动选择好好做题。

### 3.3 关键动画设计

#### 3.3.1 战斗场景动画

- **入场动画**：角色从左侧跑入，怪物从右侧跳出，各自摆出战斗姿势
- **攻击动画**：答对题目时角色冲刺到怪物面前，挥武器，屏幕震动+数字飞溅
- **受击动画**：怪物被击中时闪烁红色+后退+伤害数字飘出
- **连击特效**：连续答对时，背景颜色渐变为火焰色，数字越来越大，第 5 连击触发全屏闪光
- **暴击特效**：暴击时屏幕变暗→角色蓄力→屏幕爆裂效果+大字体伤害数字
- **怪物死亡**：怪物倒地→化为星光粒子→掉落宝箱→宝箱打开星星飞出
- **答错/超时**：怪物冲向角色→角色被击退→护盾碎裂动画

#### 3.3.2 地图动画

- **路径连线**：关卡节点之间用虚线路径连接，已通过的路径变为实线+流光效果
- **角色行走**：选择关卡时角色沿路径走到该节点
- **区域主题**：语文区域有飘落的竹叶，数学区域有浮动的数字符号，英语区域有字母气泡

#### 3.3.3 微交互

- **按钮**：hover 时轻微放大+发光，点击时下沉+涟漪效果
- **数字变化**：星星/经验变化时数字跳动（如 100→105 逐帧递增）
- **页面切换**：滑动/缩放过渡，而非硬切
- **成就解锁**：全屏撒花+徽章从灰变亮+3D 翻转

### 3.4 技术实现方案

| 技术点 | 方案 |

## 4. 副本玩法：5 种辅助玩法

### 4.1 副本总览

| 副本 | 类型 | 解锁条件 | 奖励 | 定位 |
|---|---|---|---|---|
| 五子棋挑战 | 策略对弈 | 完成 Day 5 主线 | 星星+成就 | 逻辑思维训练 |
| 百科小博士 | 知识问答 | 完成 Day 10 主线 | 星星+碎片 | 拓展知识面 |
| 记忆翻翻乐 | 记忆配对 | 完成 Day 3 主线 | 星星+宠物饲料 | 记忆力训练 |
| 速算大闯关 | 限时竞速 | 完成 Day 7 主线 | 星星+装备碎片 | 计算速度训练 |
| 词语接龙 | 语言游戏 | 完成 Day 15 主线 | 星星+碎片 | 词汇量训练 |

### 4.2 五子棋挑战

**玩法**：孩子与 AI 对弈，棋盘 15×15，AI 难度分 3 级（简单/普通/困难）。

**规则**：
- 黑子先手，五子连珠获胜
- 每局获胜奖励 5 星星
- 连胜 3 局解锁"棋圣"成就
- AI 难度随孩子胜率动态调整

**技术实现**：
- 棋盘用 CSS Grid 绘制
- AI 使用极小化极大算法（Minimax）+ Alpha-Beta 剪枝，搜索深度 3-4 层
- 落子动画：棋子从上方掉落+弹跳

### 4.3 百科小博士

**玩法**：每题 4 个选项，涵盖天文、地理、历史、科学、体育等领域，10 题一局。

**题库设计**：
- 静态题库 200+ 题，按领域分类
- 所有题目必须符合 **小学 4-5 年级认知水平**：不涉及政治敏感话题、暴力恐怖内容、成人化社会议题；以自然常识、安全知识、地理人文、科学入门为主
- 每局随机抽取 10 题，不重复
- 示例题目：
  - "太阳系中最大的行星是？"（木星）
  - "长城全长约多少公里？"（约 21,196 公里）
  - "水的化学式是？"（H₂O）
  - "人体最大的器官是？"（皮肤）
  - "中国有多少个民族？"（56 个）
  - "一年有多少天？365 天，但闰年是多少天？"（366 天）
- **题库内容红线**（以下类型不允许入库）：
  - 涉及政权、战争、宗教评价的题目
  - 名人负面新闻、社会恶性事件
  - 成人化两性内容
  - 超出小学理解范围的专业术语（如量子力学、基因工程细节）
  - 暗示或鼓励危险行为的题目（如"从多高跳下来会骨折"）

**奖励**：
- 答对 1 题得 1 星星
- 全对额外奖励 5 星星 + 随机碎片
- 累计答对 100 题解锁"百科全书"成就

### 4.4 记忆翻翻乐

**玩法**：4×4 或 4×6 的卡牌矩阵，每次翻两张，配对成功保留，全部配对完成过关。

**卡牌内容**：
- 数学科：算式与结果配对（如 "3×7" ↔ "21"）
- 语文科：成语与释义配对、上下句配对
- 英语科：单词与中文意思配对

**奖励**：
- 按用时和翻牌次数评级：S/A/B/C
- S 级奖励 8 星星 + 宠物饲料
- 累计 S 级 10 次解锁"过目不忘"成就

### 4.5 速算大闯关

**玩法**：限时 60 秒，屏幕不断出现算式，孩子快速输入答案，答对继续，答错扣 3 秒。

**难度设计**：
- 简单：一位数加减、乘法口诀内
- 普通：两位数加减、简单乘除
- 困难：混合运算、分数小数

**奖励**：
- 按答对数量评级：30+题 S 级（10 星星）、20+题 A 级（7 星星）、10+题 B 级（4 星星）
- 连续 3 次 S 级解锁"闪电侠"成就
- 排行榜记录个人最佳

### 4.6 词语接龙

**玩法**：系统给出一个词语，孩子用最后一个字作为开头接新词，AI 判断是否合法。

**规则**：
- 每局 10 轮，接对得 1 星星
- 接不上可以跳过（每局限 2 次）
- 词语必须是常见中文词汇（AI 判断）
- 示例：春天→天空→空气→气球→球赛→赛车→车站→站台→台灯→灯光

**奖励**：
- 10 轮全对奖励 10 星星 + 随机碎片
- 累计接龙 100 词解锁"词汇大师"成就

**技术实现**：
- 内置 5000+ 常见词语词典
- AI 判断词语合法性（无需调用模型，本地词典匹配）

## 5. 架构升级：本机服务 + H5 跨端

### 5.1 升级背景

V1.0 采用纯前端 PWA 架构，所有数据存储在浏览器 IndexedDB 中，孩子只能在家长电脑上玩游戏。V2.0 升级为 **本机服务 + H5 跨端** 架构，实现以下核心变化：

1. **电脑端作为家庭服务器**：家长电脑运行 Node.js 本地服务，统一管理游戏数据和业务逻辑
2. **家长端仍在电脑端运行**：通过浏览器访问 localhost，体验不变，管理功能更强大
3. **孩子端改为 H5 跨端页面**：手机、平板等任何设备通过局域网 WiFi 访问，无需安装 App
4. **数据集中存储**：从 IndexedDB（浏览器内）迁移到 SQLite（服务端），支持多设备共享同一份数据

### 5.4 数据层迁移：IndexedDB → SQLite

#### 5.4.1 迁移策略

| 阶段 | 内容 |
|---|---|
| 首次启动 | 自动检测 IndexedDB 中是否有旧数据，如有则弹出迁移确认 |
| 迁移执行 | 读取 IndexedDB 全部数据 → 转换为 SQLite schema → 写入 → 验证行数一致 |
| 迁移后 | IndexedDB 数据保留不删除（作为备份），后续读写全部走 SQLite |
| 新用户 | 直接初始化 SQLite 空库，跳过迁移 |

#### 5.4.2 数据模型映射

IndexedDB 的 13 个 Object Store 全部映射为 SQLite 表，保持字段对应关系不变，字符 ID 均使用 TEXT 类型。迁移工具 `server/migration.ts` 负责一对一转换。

### 5.7 多设备支持

架构升级天然支持多设备同时访问：

- **多孩子场景**：同一家庭多个孩子各自用自己手机/平板，选择自己的档案登录
- **数据隔离**：每个 Profile 独立存储进度，互不干扰
- **星星共享**：家庭星星池统一管理，所有孩子共享兑换同一个奖励池
- **并发安全**：SQLite 的 WAL 模式支持多读单写，家庭场景足够

### 5.8 离线降级策略

| 场景 | 策略 |
|---|---|
| 手机断网（WiFi 断开） | 显示"连接中断"提示，保留已加载题目可继续答题，结果缓存到 localStorage，恢复连接后自动同步 |
| 服务端未启动 | 孩子端显示"请先打开电脑上的学科小勇士"提示 + 家长端本地可访问 localhost 管理 |
| 服务端崩溃 | 家长端可手动重启服务（npm start），孩子端自动重连 |

### 5.9 安全设计

| 层面 | 措施 |
|---|---|
| 网络隔离 | 仅监听局域网（0.0.0.0），不暴露到公网，路由器 NAT 天然隔离 |
| 家长 PIN | 孩子端切换档案、查看家长设置需输入 PIN 码验证 |
| API 鉴权 | 孩子端 API 仅能操作自己的档案数据；管理类 API 仅 localhost 可调用 |
| 数据备份 | 家长端提供一键备份 SQLite 文件到本地，支持导出 JSON |

## 7. 风险与注意事项

1. **AI 出题成本**：1350 题的 AI 生成成本需要评估，建议首次生成用 GPT-4o-mini（低成本），后续补充也用小模型
2. **题目质量**：自动生成的大量题目需要校验逻辑兜底，确保不出现格式错误或超纲内容
3. **性能**：战斗场景的粒子特效和动画可能影响低端设备，需要做性能降级（关闭粒子/简化动画）
4. **副本平衡性**：五子棋 AI 难度需要调优，避免孩子一直输或一直赢
5. **数据迁移**：已有用户数据需要平滑迁移到新的关卡结构
6. **SQLite 并发**：better-sqlite3 为同步 API，需确保单线程写入，WAL 模式缓解读锁冲突
7. **局域网稳定性**：依赖 WiFi 质量，需做好离线降级和自动重连
8. **移动端兼容**：不同手机浏览器的 CSS 兼容性需要充分测试（iOS Safari / Android Chrome）
